Todas las personas o colaboradores que participan en un proceso de ingeniería de software y los cuales son los encargados de finanzas, la parte de marketing del negocio, este se compone de personas que gestionan el negocio, gestionan el proyecto, equipo de desarrollo, clientes, usuarios finales, además de los analistas de software encargados de analizar la calidad del software del proyecto.

En la ingeniería de software, el producto también se entiende como los artefactos de software, esta “P” se compone algunos componentes:

  1. Documentación del proyecto
  2. Código Fuente
  3. Documentos de Pruebas
  4. Documentos de Cliente
A los productos de software, se les suele llamar también como artefactos de software.

El proyecto consiste principalmente o comprende todas aquellas actividades necesarias para generar un producto de software, además este se compone de algunas fases que generalmente se siguen al desarrollar un software:

  1. Planeación
  2. Análisis de requerimientos
  3. Diseño
  4. implementación
  5. Pruebas
  6. Mantenimiento
Aquí se definen además en detalle cada una de las fases de modelos que se van a seguir para el desarrollo de software.

Esta “P” se compone de una serie de procedimientos para poder estructurar mejor el software y llevar un modelo de calidad y ameno.
